# MySQL OJ 系统 - 考试模块技术方案

## 1. 架构设计

### 1.1 整体架构
- **前端架构**：扩展现有模块化架构，新增考试相关模块
- **存储方案**：使用 localStorage/IndexedDB 本地存储
- **数据流**：考试配置 → 题目加载 → 考试过程 → 结果存储 → 分析展示

### 1.2 核心模块
- **ExamManager**：考试管理模块，负责考试的创建、编辑、删除和查询
- **ExamSession**：考试会话模块，管理考试过程和状态
- **Timer**：倒计时模块，处理考试时间控制
- **ExamResult**：结果分析模块，计算成绩和生成分析报告
- **ExamUI**：考试界面模块，负责考试相关页面的渲染

## 2. 数据结构

### 2.1 考试配置 (Exam)
```javascript
{
  id: String,           // 考试唯一标识
  name: String,         // 考试名称
  duration: Number,     // 考试时长（分钟）
  questionCount: Number, // 题目数量
  difficulty: String,   // 难度级别 (all/easy/medium/hard)
  chapters: Array,      // 题目来源章节
  createdAt: String,    // 创建时间
  createdBy: String     // 创建者用户名
}
```

### 2.2 考试会话 (ExamSession)
```javascript
{
  id: String,           // 会话唯一标识
  examId: String,       // 关联的考试ID
  userId: String,       // 考生用户名
  startTime: String,    // 开始时间
  endTime: String,      // 结束时间
  status: String,       // 状态 (in_progress/completed)
  remainingTime: Number, // 剩余时间（秒）
  currentQuestion: Number, // 当前题目索引
  questions: Array,     // 考试题目列表
  answers: Object,      // 答案存储
  score: Number         // 最终得分
}
```

### 2.3 题目实例 (QuestionInstance)
```javascript
{
  id: String,           // 实例唯一标识
  problemId: Number,    // 原始题目ID
  order: Number,        // 题目顺序
  answered: Boolean,    // 是否已回答
  answer: String,       // 考生答案
  result: String,       // 判题结果 (AC/WT/CE/RE)
  score: Number         // 得分
}
```

## 3. 核心功能实现

### 3.1 考试管理
- **创建考试**：教师用户可以设置考试名称、时间、题目数量、难度范围和来源章节
- **编辑考试**：修改已有考试的配置
- **删除考试**：移除不需要的考试
- **查看考试列表**：显示所有可用的考试

### 3.2 考试参加
- **开始考试**：学生用户选择考试并开始，系统随机生成题目
- **考试过程**：显示倒计时、题目内容、答案输入区域
- **题目导航**：支持在题目间切换，显示题目状态
- **自动保存**：定期保存考试进度，防止数据丢失
- **自动提交**：时间结束时自动提交答案

### 3.3 评分与结果
- **自动评分**：使用现有判题引擎对答案进行评分
- **成绩计算**：根据题目得分计算总分
- **结果分析**：生成详细的考试分析报告
- **历史记录**：存储考试历史记录，支持查询

### 3.4 权限控制
- **教师权限**：可以创建、编辑、删除考试
- **学生权限**：只能参加考试，不能管理考试

## 5. 技术实现细节

### 5.1 存储实现
- **考试配置**：使用 localStorage 存储，格式为 JSON
- **考试会话**：使用 IndexedDB 存储，支持更大数据量
- **历史记录**：使用 IndexedDB 存储，限制单用户 100 条记录

### 5.2 时间管理
- **倒计时实现**：使用 `setInterval` 实现倒计时
- **时间同步**：定期校准时间，防止页面切换导致的时间偏差
- **时间结束处理**：触发自动提交，显示考试结束提示

### 5.3 题目生成
- **随机选择**：使用 Fisher-Yates 算法随机选择题目
- **题目过滤**：根据考试配置筛选符合条件的题目
- **题目加载**：懒加载题目数据，提高性能

### 5.4 防作弊措施
- **页面锁定**：禁止考试期间刷新页面
- **时间监控**：检测异常的时间行为
- **答案保护**：防止查看其他题目的答案

### 5.5 性能优化
- **资源预加载**：提前加载考试所需资源
- **缓存策略**：缓存已加载的题目和配置
- **防抖处理**：减少频繁的存储操作
- **状态管理**：优化考试状态的管理和更新

## 9. 风险评估

### 9.1 技术风险
- **时间同步**：用户设备时间可能不准确
- **存储限制**：本地存储容量有限
- **浏览器兼容性**：不同浏览器对 Web Storage 的支持差异
- **数据安全**：本地存储数据可能被用户篡改

### 9.2 功能风险
- **题目随机性**：确保题目选择的公平性
- **评分准确性**：确保自动评分的准确性
- **防作弊**：防止考试期间的作弊行为
- **数据丢失**：防止考试数据意外丢失

### 9.3 应对措施
- **时间校准**：使用服务器时间或网络时间协议校准
- **数据压缩**：优化数据存储格式，减少存储空间
- **兼容性处理**：添加浏览器兼容性检测和降级方案
- **数据加密**：对敏感数据进行加密存储
- **自动保存**：定期自动保存考试进度
